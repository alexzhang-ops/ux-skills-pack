# 資訊架構基礎

> 供 ux-empathize skill 的 Step 5（Constraint Inventory）參考。在研究階段同步思考資訊架構，為設計階段打好基礎。

## 資訊架構流程

```
內容盤點 → 使用者中心分類 → 層級定義 → 導航設計 → 驗證
```

### 1. 內容盤點（Content Inventory）

列出功能涉及的所有資訊類型：

| 問題 | 目的 |
|------|------|
| 這個功能有哪些資訊？ | 確定範圍 |
| 使用者最常需要哪些？ | 決定優先級 |
| 哪些資訊之間有關聯？ | 決定分組 |
| 哪些是必填/必看，哪些是選填/輔助？ | 決定顯示層級 |

### 2. 使用者中心分類

**原則**：用使用者的思維模式分類，不是用組織架構分類。

| ❌ 組織思維 | ✅ 使用者思維 |
|---|---|
| 按部門分（人事、財務、業務） | 按任務分（查詢、建立、審核） |
| 按資料類型分（訂單、商品、會員） | 按工作流程分（接單→備貨→出貨→結帳） |
| 按系統模組分 | 按使用頻率排序 |

### 3. 層級控制

| 原則 | 說明 |
|------|------|
| **≤3 層深度** | 從入口到完成任務最多 3 次點擊/切換 |
| **寬度 > 深度** | 同一層級多選項 > 多層級少選項 |
| **高頻任務淺層化** | 每天用的功能在第 1 層就要能到達 |
| **低頻任務可藏深** | 設定、匯出等偶爾用的功能可以在第 2-3 層 |

## 導航模式選擇

| 模式 | 適用場景 | 平台適用 |
|------|---------|---------|
| **Tab Bar** | ≤5 個頂層功能，行動裝置 | iPhone / Android |
| **Side Tab Bar** | ≥5 個頂層功能，大螢幕 | iPad / Desktop |
| **Sidebar** | 多層級導航，內容管理 | iPad / Web / Desktop |
| **Bottom Navigation** | 行動優先，3-5 個核心功能 | iPhone / Android |
| **Drawer** | 次要導航，空間有限 | 所有平台 |
| **Breadcrumb** | 多層級內容，使用者需要知道位置 | Web / Desktop |
| **Segmented Control** | 同一頁面的 2-5 個檢視模式切換 | 所有平台 |

## 驗證方法

### 卡片分類法（Card Sorting）
- **開放式**：讓使用者自己分組和命名 → 發現使用者的心智模型
- **封閉式**：給定分類，讓使用者排列 → 驗證你的分類是否直覺
- 線上工具：Optimal Workshop、UserZoom

### 樹狀測試（Tree Testing）
- 給使用者一個任務（例如「找到退款紀錄」），看他在導航樹中走哪條路
- 驗證導航結構是否符合使用者預期
- 成功率 ≥80% = 結構合理；<60% = 需要調整

## Anti-Patterns

| Anti-pattern | 問題 | 替代做法 |
|---|---|---|
| 用組織架構當 IA | 使用者不在乎公司怎麼分部門 | 按任務和使用頻率組織 |
| 忽略任務頻率 | 常用功能藏太深 | 高頻 = 淺層，低頻 = 可深層 |
| 層級太深 | 每次操作要點 5 次才到目的地 | 控制在 ≤3 層 |
| 分類太細 | 10 個只有 2-3 項的分類 | 合併相關分類，減少認知負擔 |
| 命名不直覺 | 用系統術語而非使用者語言 | 用使用者訪談中的詞彙命名 |
