# 線框圖方法論

> 供 ux-design skill 的 Step 3（Generate Alternatives）參考。

## 保真度光譜

| 保真度 | 目的 | 形式 | 適用時機 |
|--------|------|------|---------|
| **Low** | 驗證結構合理性 | 文字描述、手繪草圖、ASCII | 剛開始探索，需要快速比較多個方案 |
| **Mid** | 驗證互動流程 | 結構化線框圖 + 互動標註 | 方向確定後，細化佈局和互動 |
| **High** | 視覺展示 / 利益關係人溝通 | 接近最終的視覺設計 | 設計確認、開發交付 |

**ux-design skill 通常在 Mid 保真度**：用 ASCII wireframe + 互動說明產出替代方案。

## 頁面骨架建立步驟

### 1. 定義目標
- 這個頁面要幫使用者完成什麼任務？
- 主要行動（Primary Action）是什麼？
- 使用者進入這個頁面時的心理狀態？（急著找東西 vs 瀏覽探索）

### 2. 列出內容
- 這個頁面需要顯示哪些資訊？
- 哪些是必要的，哪些是輔助的？
- 哪些資訊使用者會最先想看到？

### 3. 安排佈局
- 最重要的內容放最顯眼的位置（左上 / 頂部）
- 主要行動按鈕放在視線動線的終點
- 相關資訊分組（Proximity 原則）
- 考慮平台特性（iPad 寬螢幕可用雙欄）

### 4. 標註互動
- 點擊/觸碰後發生什麼？
- 狀態變化：Normal → Loading → Success / Error
- 導航流向：這個頁面如何進入？完成後去哪裡？

### 5. 審查
- 能否在 3 步內完成主要任務？
- 是否有不必要的元素可以移除？
- 使用者是否能立即理解該怎麼操作？

## 佈局模式庫

### 列表式（List）
```
┌─────────────────────────┐
│ 搜尋 / 篩選列           │
├─────────────────────────┤
│ ┌─────────────────────┐ │
│ │ Item 1              │ │
│ ├─────────────────────┤ │
│ │ Item 2              │ │
│ ├─────────────────────┤ │
│ │ Item 3              │ │
│ └─────────────────────┘ │
└─────────────────────────┘
```
**適用**：訂單列表、會員列表、搜尋結果

### 卡片網格（Card Grid）
```
┌─────────────────────────┐
│ 分類選擇列               │
├────────┬────────┬───────┤
│ Card 1 │ Card 2 │ Card 3│
├────────┼────────┼───────┤
│ Card 4 │ Card 5 │ Card 6│
└────────┴────────┴───────┘
```
**適用**：商品目錄、服務選擇、Dashboard

### 側欄 + 內容（Sidebar + Content）
```
┌──────┬──────────────────┐
│      │                  │
│ Side │    Content       │
│ bar  │                  │
│      │                  │
└──────┴──────────────────┘
```
**適用**：iPad 主導航、設定頁、多層級內容

### Drawer 面板（Right-side Panel）
```
┌──────────────┬──────────┐
│              │          │
│   Main       │  Drawer  │
│   Content    │  Panel   │
│              │  (400pt) │
│   (dimmed)   │          │
└──────────────┴──────────┘
```
**適用**：詳情面板、付款流程、篩選面板

### Tab 切換
```
┌─────────────────────────┐
│ [Tab A] [Tab B] [Tab C] │
├─────────────────────────┤
│                         │
│   Tab A Content         │
│                         │
└─────────────────────────┘
```
**適用**：檢視模式切換、資訊分類顯示

## 複雜度觸發器

以下情境需要 **≥2 個替代方案**：

| 觸發器 | 為什麼 |
|--------|--------|
| B2B Dashboard | 資訊密度高，佈局方式差異大 |
| 多角色使用 | 不同角色看到不同內容/權限 |
| 數據密集 | 表格 vs 卡片 vs 圖表的取捨 |
| 多步驟流程 | 步驟式 vs 單頁長表單 vs 分區塊 |
| 首次使用 vs 老手 | Onboarding 引導 vs 效率優先 |

## Anti-Patterns

| Anti-pattern | 問題 | 替代做法 |
|---|---|---|
| 過度精緻的線框圖 | 利益關係人以為是最終設計 | 保持低保真，用灰色/方框代替真實內容 |
| 缺少任務流程 | 只畫靜態頁面，沒考慮頁面間的關係 | 標註「從哪來→做什麼→去哪裡」 |
| 層級扁平化 | 所有資訊等重要性 | 用大小、粗細、色彩建立視覺層級 |
| 直接套用範本 | 不考慮具體需求 | 範本只是起點，需根據任務和使用者調整 |
