# 互動設計模式

> 供 ux-design skill 的 Step 6（HTML Prototype）參考。

## 狀態機思維

每個互動元件都有狀態。設計時應覆蓋所有狀態：

### 按鈕狀態
| 狀態 | 視覺表現 | 觸發條件 |
|------|---------|---------|
| Normal | 預設樣式 | 初始狀態 |
| Hover | 微亮/陰影加深 | 游標移入（iPad 支援 pointer） |
| Pressed | 縮放 0.95 / 色彩加深 | 觸碰/點擊中 |
| Disabled | 灰色、降低透明度 | 條件不滿足 |
| Loading | Spinner 或文字變「處理中…」 | API 呼叫中 |

### 輸入框狀態
| 狀態 | 視覺表現 | 觸發條件 |
|------|---------|---------|
| Empty | Placeholder 文字 | 初始狀態 |
| Focused | 邊框變色（品牌色） | 使用者點擊/Tab 進入 |
| Filled | 顯示輸入值 | 有值且未 focus |
| Error | 紅色邊框 + 錯誤訊息 | 驗證失敗 |
| Disabled | 灰色背景 | 不可編輯 |

### 頁面狀態
| 狀態 | 顯示內容 | 設計要點 |
|------|---------|---------|
| Loading | Skeleton screen | 保持佈局結構，灰色色塊動畫 |
| Empty | 插圖 + 說明 + CTA | 引導使用者下一步行動 |
| Normal | 實際內容 | 主要設計 |
| Error | 錯誤說明 + 重試按鈕 | 明確告知原因和解決方式 |
| Partial | 部分內容 + 載入更多 | 漸進式載入 |

## 回饋設計

### 回饋時機
| 時機 | 方式 | 範例 |
|------|------|------|
| **即時** (<100ms) | 視覺變化 | 按鈕按下效果、Toggle 切換 |
| **短暫** (<1s) | 骨架屏/進度條 | 頁面載入、資料搜尋 |
| **中等** (1-10s) | 進度指示 + 可取消 | 檔案上傳、批次操作 |
| **長時間** (>10s) | 背景處理 + 通知 | 匯出報表、資料同步 |

### 回饋類型
- **視覺回饋**：色彩變化、動畫、icon 切換 — 最常用
- **觸覺回饋**：震動（haptic feedback）— 適用確認操作
- **文字回饋**：Toast / Snackbar / Alert — 需要傳達具體訊息時
- **音效回饋**：嗶聲 — 掃描、結帳等特定場景

## 導航模式

### iPad 導航（Side Tab + Content）
```
┌──────┬───────────────────────────┐
│      │ ← Title        Actions   │
│ Tab  ├───────────────────────────┤
│ Bar  │                          │
│      │      Page Content         │
│      │                          │
│ [日曆]│                          │
│ [預約]│                          │
│ [訂單]│                          │
│      │                          │
│ ─────│                          │
│ [LOGO]│                          │
└──────┴───────────────────────────┘
```
- Side Tab 可收合（寬螢幕展開、窄螢幕 icon-only）
- 子頁面用 NavigationController push
- 詳情/操作面板用 Right-side Drawer

### 頁面間導航
| 動作 | 方式 | 動畫 |
|------|------|------|
| 進入子頁 | Push（NavigationController） | 從右滑入 |
| 開啟面板 | Present（右側 Drawer） | 從右滑入 + 遮罩 |
| 確認/警告 | Modal（formSheet） | 淡入 + 遮罩 |
| 返回 | Back button / 滑動手勢 | 向右滑出 |
| 關閉面板 | X 按鈕 / 點遮罩 / 滑動 | 向右滑出 |

## 表單設計

### 驗證策略
| 策略 | 時機 | 適用 |
|------|------|------|
| **即時驗證** | 輸入離開（onBlur） | Email 格式、必填欄位 |
| **輸入中驗證** | 每次按鍵（onChange） | 密碼強度、字數限制 |
| **提交驗證** | 按下送出 | 伺服器端驗證、跨欄位驗證 |

### 錯誤訊息原則
- **位置**：緊鄰相關輸入框下方（不要用 Alert 彈窗）
- **語氣**：說明「怎麼修正」而非「你做錯了」
  - ❌「格式錯誤」
  - ✅「請輸入有效的 Email 地址，例如 name@example.com」
- **時機**：不要在使用者還沒打完就顯示錯誤

### 步驟式表單 vs 單頁表單
| 類型 | 適用 | 優點 | 缺點 |
|------|------|------|------|
| 步驟式 | ≥5 個欄位、有條件分支 | 不嚇人、可漸進揭露 | 使用者不知道還有幾步 |
| 單頁 | ≤5 個欄位、簡單表單 | 一目瞭然、快速完成 | 欄位多時壓迫感 |
| 分區塊 | 中等複雜度 | 結構清晰、可摺疊 | 需要良好的區塊命名 |

## 手勢設計（iPad）

| 手勢 | 用途 | 設計注意 |
|------|------|---------|
| **Tap** | 選擇、確認 | 觸控目標 ≥44×44pt |
| **Long Press** | 更多選項、預覽 | 需要提示（例如 context menu icon） |
| **Swipe Left/Right** | 快捷操作（刪除/編輯） | 提供可見的替代方式（按鈕） |
| **Pinch** | 縮放（地圖、圖片） | 只在有意義的地方啟用 |
| **Drag-and-Drop** | 排序、移動 | 有明確的拖曳手柄、放置目標高亮 |
| **Pointer Hover** | 預覽、工具提示 | iPad 支援滑鼠/觸控板 hover |

## 危險操作保護

| 操作風險 | 保護方式 |
|---------|---------|
| 低（切換 Toggle） | 無需確認，可即時撤銷 |
| 中（離開未儲存頁面） | 提示「你有未儲存的變更」 |
| 高（刪除資料） | 確認對話框 + 明確的破壞性按鈕（紅色） |
| 極高（批次刪除/不可復原） | 二次確認（輸入文字確認）或冷卻時間 |
